# 템플릿 전용 스레드 시작 프롬프트

아래 문장을 새 스레드 첫 메시지로 붙여넣어 주세요.

```
이 스레드는 템플릿 전용입니다.
결제/주문/알림/인증 작업은 하지 말고, 템플릿(Figma→JSON→앱 적용)만 진행하세요.
고정 규격은 정사각형 1440x1440, 세로 1080x1440, 가로 1440x1080 입니다.
운영 기준은 portrait master 1세트 + variants(square/landscape) 우선이며, QA에서 깨질 때만 비율별 Figma 원본을 추가하세요.
strictLayout=true, autoFit=false를 기본으로 보되, variants 보정이 더 중요하면 그 기준을 우선하세요.
예시 이미지는 반드시 포함하고, 최종 운영 이미지는 Figma 승인본을 export 해서 CDN URL로 관리하세요. figma.com/api/mcp/asset URL을 최종 JSON에 직접 쓰면 안 됩니다.
별도 요청이 없어도 플레이스홀더만 만들고 멈추지 말고, 실제 예시 이미지 삽입과 대표 미리보기 검수까지 기본으로 이어서 완료하세요.
플레이스홀더 색 박스 상태나 빈 이미지 슬롯 상태는 완료로 보지 마세요.
로컬 asset은 fallback 또는 개발 검수용으로만 유지하세요.
커버에는 배경 이미지 또는 배경 비주얼을 반드시 넣고, 첫 화면에서 주제와 무드가 바로 읽히게 하세요.
페이지 수는 최소 12장이지만, 상품성 기준 권장은 14~20장입니다.
커버와 페이지 레이아웃은 서로 달라야 하며, 풀블리드/포토 프레임/타이포 중심/정보형/엔드형 등으로 리듬을 만들어 주세요.
사진은 장식이 아니라 실제로 눈에 들어와야 하며, 전체 세트는 배경색/타이포/포인트 컬러/사진 무드가 일관된 아트디렉션으로 묶여야 합니다.
미리보기/상세/스텝2/편집기/읽기화면에서 1:1 일치 검수까지 완료하세요.
스토어 카드/상세 커버/페이지 미리보기는 Figma 승인 export 이미지를 우선 사용하고, templateJson 재렌더는 fallback 으로만 사용하세요.
coverImageUrl 과 previewImages[0] 는 반드시 같아야 하며, previewImages 는 최소 4장 이상 유지하세요.
CDN 업로드 후 실제 이미지 디코드 검증까지 통과한 파일만 최종 반영하세요.
```
